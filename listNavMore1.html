Before you panic, media queries don't have to be overwhelming if you start simple and keep some important tips in mind.

Very Basic Media Terminology

REPONSIVE DESIGN - The position and size of the elements on the page change to suit the viewport size and orientation.

MEDIA QUERY - Media query is a technique introduced in CSS3 that uses a media rule to identify a block of CSS properties that will be used only if a certain condition or conditions are true.

BREAKPOINT - the identified point at which a media query starts or ends. Often represented by minimum width in pixels or maximum width in pixels.

VIEWPORT - The viewport is the visible area of a webpage, and it varies by device and screen size.

MEDIA RULE - A media rule holds within brackets all of the style rules that relate to the specific condition being met. In the case of a simple media query for a mobile first website, it will likely be defined by a minimum or a maximum width.

Device Dilemna

No device dilemna needed. Don't think about devices when determining your media queries, instead make your page look good as it goes from a small width to a large width. Eventually, some of your breakpoints will do a little, and some will do a lot.

Bits

Line height defines the total height of the line box. Called leading in typographical terms, we set line-height with a unitless number. If we set line-height at 1.5, that means 1.5 times the current font size. In CSS, leading is applied to the top and the bottom of the line boxes. If the font is default 16px/1rem and the line-height is 1.5 (24px) 24-26=8/2=4 so 4px or .25rem applied to top and bottom. Line height for responsive averages 1.5 for desktop and 1.3 for the smallest screens.

Add box-sizing: border-box declaration globally to keep you from having to worry about the box model.

Setting up your basic viewport meta data in the head helps control the scale in portrait or landscape.

Language of your page. Add lang ="en" to your html opening tag.

display:inline-block takes advantage of both inline and block. With just the inline, you get the horizontal display and you can add horizontal margins, paddings and borders, but adding vertical margins, padding, borders and widths or heights will do nada. You need inline block for all that tweaking ability.

Step by Step Basic Media Query

  1. Design mobile first
  2. Determine the differences needed for a media query.
  3. Create a media query with a minimum width.
  4. Create style rules that include only the differences.
    1. Your global styles should ideally consider mobile first. Think simple, everything including navigation can go the full width, and flow one thing after another in logical order. Example page one
    2. At the bottom of your global styles, add a media query [ @media (min-width: px) {} ] with a minimum width that will look good with your content and navigation. Add style rules only for what needs to change. Example page two
    3. Now your navigation needs to be addressed. Inline block and widths for list elements is generally an easy solution. One helpful trick to making inline block navigation behave is to zero out the font size at ul, the level above list, or if you don't have a list, at the level above a or nav. Then in the li or if you don't have a list at the a, add the font-size back. This will take away the spaces between inline block elements Example page three.
    4. One of the differences might be elements that will be side by side in larger viewports. You'll need to add widths to those components appropriately, but don't add more than just what needs to change. The main and aside sections that were 100% width in mobile view will be 40% and 60% for this page. Example page four.
    5. Finally, add the appropriate float and don't forget to clear so that your footer will stay in place. Example page five.
  5. Be sure all media query styles are inside the media query brackets.
